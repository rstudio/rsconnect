# Deploy a single document

Deploys a single R Markdown, Quarto document, or other file (e.g.
`.html` or `.pdf`).

When deploying an `.Rmd`, `.Qmd`, or `.html`, `deployDoc()` will attempt
to automatically discover dependencies using
[`rmarkdown::find_external_resources()`](https://pkgs.rstudio.com/rmarkdown/reference/find_external_resources.html),
and include an `.Rprofile` if present. If you find that the document is
missing dependencies, either specify the dependencies explicitly in the
document (see
[`rmarkdown::find_external_resources()`](https://pkgs.rstudio.com/rmarkdown/reference/find_external_resources.html)
for details), or call
[`deployApp()`](https://rstudio.github.io/rsconnect/dev/reference/deployApp.md)
directly and specify your own file list in `appFiles`.

Supported servers: All servers

## Usage

``` r
deployDoc(doc, ..., logLevel = c("normal", "quiet", "verbose"))
```

## Arguments

- doc:

  Path to the document to deploy.

- ...:

  Additional arguments to
  [`deployApp()`](https://rstudio.github.io/rsconnect/dev/reference/deployApp.md).
  Do not supply `appDir`, `appFiles`, or `appPrimaryDoc`; these three
  parameters are automatically generated by `deployDoc` from the
  document.

- logLevel:

  One of `"quiet"`, `"normal"` or `"verbose"`; indicates how much
  logging to the console is to be performed. At `"quiet"` reports no
  information; at `"verbose"`, a full diagnostic log is captured.
